# PromptShare Live 서비스 사용 안내서

PromptShare Live는 강사가 학생들에게 실시간으로 프롬프트를 전달하고 관리할 수 있는 가볍고 안전한 웹 서비스입니다. 

### 서버 실행 및 초기 접속

**서버 실행 방법**
터미널(또는 명령 프롬프트)에서 프로젝트 폴더로 이동한 뒤 아래 명령어를 입력하여 서버를 실행합니다.
```bash
node server.js
```

**초기 정보 확인**
서버가 정상적으로 실행되면 터미널 화면에 아래와 같이 초기 접속 랜더링 URL과 관리자(강사) 비밀번호가 출력됩니다. (기본 비밀번호: `1234`)
```text
서버가 시작되었습니다. http://localhost:3000
현재 접속 URL 경로: /live/abcd1234
관리자 비밀번호: 1234
```
출력된 접속 URL로 브라우저를 통해 접속하면 서비스 화면을 볼 수 있습니다.

### 강사 (Host) 사용법

강사는 서비스 내에서 고유 권한을 획득하여 프롬프트 전송, 삭제, 및 교육장 URL 갱신 등을 관리할 수 있습니다.

**관리자 로그인**
* 화면 우측 상단의 **자물쇠 아이콘(🔒)**을 클릭합니다.
* 터미널에 표시된 관리자 비밀번호를 입력하고 **방 입장**을 클릭합니다.
* 인증이 완료되면 화면 상단에 '관리자 모드' 배지와 함께 관리자 전용 도구 패널이 나타납니다.

**프롬프트 전송 (마크다운 지원)**
* 하단의 입력창에 전달할 프롬프트 내용을 작성합니다. (마크다운 문법 적용 가능)
* **전송** 버튼을 누르면 접속 중인 모든 학생의 화면에 실시간으로 프롬프트가 나타납니다.

**학생 초대 (접속 링크 공유)**
* 화면 상단 **초대 링크 복사** 버튼을 누르면 현재 유효한 교육장 전체 URL이 복사됩니다.
* 복사된 링크를 학생들에게 공유하여 접속하도록 안내합니다.

**프롬프트 관리 및 보안**
* **개별 삭제**: 잘못 전송된 프롬프트나 숨기고 싶은 프롬프트는 각 메시지 우측의 **삭제** 버튼을 눌러 학생 화면에서도 즉시 지울 수 있습니다.
* **전체 초기화**: **대화 전체 초기화** 버튼을 누르면 화면에 전송되었던 모든 프롬프트 내역이 삭제됩니다.
* **접속 경로 갱신**: **접속 경로 갱신** 버튼을 누르면 현재 룸의 URL이 새로운 무작위 경로로 바뀝니다. 이전 생성된 URL로 접속하던 학생들은 모두 연결이 차단되므로, 수업을 새로 시작하거나 보안을 위해 외부 인원의 접근을 막을 때 유용합니다. (갱신 후 '초대 링크 복사'를 통해 새 주소를 공유해야 합니다.)

### 학생 (Guest) 사용법

학생은 강사가 발급해 준 고유 URL을 통해서만 교육장에 입장할 수 있으며, 권한 없이 별도로 로그인할 필요가 없습니다.

**교육장 입장 및 프롬프트 수신**
* 강사가 공유해 준 링크(예: `http://localhost:3000/live/abcd1234`)를 브라우저 주소창에 입력하여 접속합니다.
* 강사가 프롬프트를 전송하면 새로고침 없이 화면에 실시간으로 프롬프트가 표시됩니다.

**프롬프트 복사**
* 필요한 프롬프트 박스 우측의 **복사** 버튼을 누르면 내용이 클립보드에 바로 저장되어 실습에 활용할 수 있습니다.

**연결 해제 시 대응**
* 화면에 "확인된 교육장 주소가 변경되었습니다"라는 붉은색 경고 메시지가 나타난다면, 강사가 보안을 위해 유효 접속 주소를 갱신한 것입니다. 이 경우 이전 주소로는 더 이상 프롬프트를 받을 수 없으므로, 강사에게 새로운 URL을 다시 안내받아 재접속해야 합니다.

### 자동 종료 (데이터 관리)

* **6시간 무활동 초기화**: 시스템 자원 관리를 위해, 강사의 마지막 프롬프트 전송이나 활동 시점으로부터 **6시간 동안 아무런 활동이 없을 경우** 서버 메모리의 대화 내용이 모두 영구 삭제되고 교육장 주소도 새롭게 초기화됩니다.
* 서버가 완전히 재시작될 경우에도 모든 대화 내용은 삭제되므로 휘발성 기록장으로 사용하는 것을 권장합니다.
